# شرح نیازمندی‌های نهایی

## فراخوانی کامند

- publisher confirmation
- حتما مطمئن شیم که ارسال میشه
- اگه صفی براش وجود نداشت برای مدت معینی پیام توی صف بمونه (ttl)
- پیام بین گیرنده‌ها round robin بشه. مگه در حالت‌های استثنا که چند گیرنده با هم دریافت کنن.
- امکان ارسال ack
- امکان تعریف priority
- برای زمان مشخصی باید منتظر پاسخ بمونه. بعد از اون دیگه جواب ارزش نداره! اگه پیام هنوز توی صف باشه، باید بتونیم پیام رو از توی صف حذف کنیم.
- اگه بعد از مدت زمان مشخص شده برای دریافت جواب، جواب اومد، باید یه لاگ خطا بندازه.

## انتشار و دریافت ایونت

- یک فرستنده و چند گیرنده
- receiver: routing key + round robin چند گیرنده و بین چند گیرنده روند رابین
- publisher confirmation
- امکان ارسال ack
- اگه صفی برای دریافت نبود، نیاز به ارسال نیست!

## request/reply

دو استفاده اصلی داره:

- فراخوانی کامند
- خوندن لیست

- ممکنه چند جواب داشته باشه. حداقل دو جواب، جواب عملیات موفق و خطا در انجام عملیات
- برای زمان مشخصی باید منتظر پاسخ بمونه. بعد از اون دیگه جواب ارزش نداره! اگه پیام هنوز توی صف باشه، باید بتونیم پیام رو از توی صف حذف کنیم.
- اگه بعد از مدت زمان مشخص شده برای دریافت جواب، جواب اومد، باید یه لاگ خطا بندازه.
- اگه فراخوانی یک کامند از طریق این روش انجام شده بود، تمام جواب‌های ممکن باید توی صف پاسخ مشخص بشن.
